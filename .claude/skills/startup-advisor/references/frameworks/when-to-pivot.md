# When to Pivot

## Pivot Signals
- **Metrics flat for 3+ months** despite active iteration
- **Users use it differently** than intended (follow them)
- **One feature gets all the love** — the product is too broad
- **CAC keeps rising** with no path to improvement
- **Founder passion is dying** — you dread working on it

## Pivot Process
1. **Diagnose** — Is it the problem, solution, market, or business model?
2. **Mine your data** — What IS working? Pivot from strength
3. **Talk to users** — Especially churned users and power users
4. **Test before committing** — Validate the new direction cheaply
5. **Communicate** — Tell investors, team, customers honestly

## Types of Pivot
- **Customer segment** — Same product, different audience
- **Problem** — Same audience, different pain point
- **Solution** — Same problem, different approach
- **Channel** — Same product, different distribution
- **Business model** — Same product, different monetization

## YC Wisdom
- Most successful startups pivoted at least once
- The best pivots preserve something from the original (team knowledge, technology, user base)
- Pivot ≠ failure. Stubbornly refusing to pivot = failure.

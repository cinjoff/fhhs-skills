  <files>.claude/skills/auto/auto-orchestrator.cjs</files>
  <action>
  Create a `StreamJsonParser` class (or simple closure) that buffers stdout chunks
  and emits complete JSON-line objects. This handles the chunk-splitting problem
  where a JSON line may arrive across multiple `data` events.

  ```
  function createJsonLineParser(onLine) {
    let buffer = '';
    return function feed(chunk) {
      buffer += chunk;
      let nlIdx;
      while ((nlIdx = buffer.indexOf('\n')) !== -1) {
        const line = buffer.slice(0, nlIdx).trim();
        buffer = buffer.slice(nlIdx + 1);
        if (!line) continue;
        try {
          const obj = JSON.parse(line);
          onLine(obj);
        } catch { /* non-JSON line — ignore */ }
      }
    };
  }
  ```

  Wire this into `runClaudeSession`'s `child.stdout.on('data')` handler (around line 835).
  For each parsed JSON object:
  - If `obj.tool` exists: call `onToolCall(obj.tool, timestamp)` callback
  - If `obj.usage` exists: accumulate tokens for live cost estimate
  - If `obj.type === 'result'`: mark session as completing

  The existing `stdout += chunk` accumulation stays — parseSessionMetrics still runs
  post-session. This is an additive real-time layer on top.
  </action>

# Plan 07-17: Auto Reliability & Tracker Visibility Fixes

## Goal

Fix three concrete bugs discovered during live /fh:auto usage:
1. Tracker shows "pending" for active sessions during planning/review wave
2. Phase names are null in step_history metrics
3. /fh:auto falls back to inline execution when FHHS_SKILLS_ROOT is stale

## Root Causes

**Bug 1 — Tracker pending for active sessions:**
- `tracker/server.cjs` checks `phase_states` for "building/planning/reviewing" only
- After the planning wave completes, states become "planned"/"rescheduled-sequential"
- `autoState.active=true` is not checked as a fallback
- Result: nerve-os showed `status: "pending"` throughout its planning wave

**Bug 2 — Phase names null in metrics:**
- All 5 `_autoStatus.stepHistory.push({...})` call sites omit `phase_name`
- Phase name IS available as `phase.name` at all sites
- Result: tracker dashboard shows `null` for phase names in step metrics

**Bug 3 — Auto falls back to inline execution:**
- `FHHS_SKILLS_ROOT` env var can become stale (e.g., points to 1.58.1 after 1.58.5 installs)
- When orchestrator not found, SKILL.md only printed an error without a hard STOP directive
- Claude interpreted the error as recoverable and proceeded to run phases inline
- Result: phases executed in main session instead of via `claude -p` subprocesses

## Changes Made

### tracker/server.cjs
- Extended `activeStatuses` set to include `'planned'`, `'reviewed'`, `'rescheduled-sequential'`
- Added `autoState.active === true` fallback: if active but no phases are detected as active yet (0 completed), force `hasActive = true`

### auto-orchestrator.cjs
- Added `phase_name: phase.name` to all 5 `_autoStatus.stepHistory.push()` call sites:
  - Sequential pipeline step handler (line ~2121)
  - Parallel planning wave (line ~2443)
  - Parallel review wave (line ~2593)
  - Parallel build wave (line ~2804)
  - Sequential fallback loop (line ~2933)

### SKILL.md Step 6
- Replaced direct `$FHHS_SKILLS_ROOT` reference with fallback discovery:
  - If primary path works → use it
  - If stale → search `~/.claude/plugins/cache/fhhs-skills/fh/*/` for latest version
  - If neither found → FATAL abort with explicit "STOP — do NOT proceed manually"
- Added `**CRITICAL:**` note after code block: on FATAL exit, stop immediately, do not do work inline
- Fixed `--check-corrections` invocation to use resolved `$ORCHESTRATOR` variable

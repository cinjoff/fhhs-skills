<task type="auto" wave="1">
  <name>Task 1: Create DECISIONS.md template and format specification</name>
  <files>.claude/skills/build/references/decisions-template.md</files>
  <action>
  Create `.claude/skills/build/references/decisions-template.md` (co-located with build skill, shipped with plugin) with:

  1. DECISIONS.md file format specification:
     - YAML frontmatter: `type: decisions`, `project: {name}`, `created: {date}`
     - `## Decision Log` section
     - Each decision as `### D-{NNN}: {title}` with fields:
       - **Phase:** phase identifier (or "project" for cross-phase decisions)
       - **Step:** which skill/step made this decision (e.g., "plan-work Step 3", "build Wave 2")
       - **Context:** what prompted the decision (1-2 sentences)
       - **Options considered:** lettered list with pros/cons
       - **Selected:** which option + checkmark
       - **Rationale:** why this option (1-2 sentences)
       - **Confidence:** HIGH / MEDIUM / LOW
       - **Affects:** list of artifacts/plans impacted
       - **Status:** ACTIVE / CORRECTED / SUPERSEDED
     - LOW confidence decisions get `⚠ NEEDS REVIEW` after the confidence field

  2. Auto-decision heuristics — when making decisions autonomously, prefer:
     - Options that match existing codebase patterns over novel approaches
     - Reversible choices over irreversible ones
     - Simpler solutions over complex ones
     - Well-documented libraries over hand-rolling
     - Options with fewer new dependencies
     - The option that keeps the most doors open for future changes

  3. Instructions for subagents on how to append decisions:
     - If DECISIONS.md does not exist, CREATE it with frontmatter first, then append
     - If DECISIONS.md exists but is corrupt/unparseable, rename to DECISIONS.md.bak, create fresh with a `## Recovery` note, then append
     - Read existing DECISIONS.md to find last D-{NNN} ID number, increment by 1
     - Append new decision block at end of file
     - Never edit or delete existing entries
     - Decision IDs are sequential — single-writer guarantee (one session at a time)

  4. Decision correction format (for human review):
     - `### D-{NNN}: {title} [CORRECTED]`
     - **Corrected by:** human / agent
     - **Original:** D-{original ID}
     - **New selection:** which option
     - **Correction rationale:** why
     - **Cascade needed:** list of affected artifacts that need updating
  </action>

  - Contains CORRECTED format

  - Contains auto-decision heuristics section
  - Contains file creation instructions (for first-run case)

Fix context flow inefficiencies in the auto pipeline. Remove the wasteful orchestrator pre-index session, fix the plan-work bootstrap to index codebase files individually for better search granularity, add a ctx_search probe in build to avoid redundant indexing, and add post-plan source file indexing so plan-review and build can access relevant source via ctx_search.

<context>
## Audit Findings (from context flow review)

**Context-mode DB is keyed by project directory, NOT session ID.**
Path: `~/.claude/context-mode/sessions/<SHA256(projectDir)[:16]>.db`
All claude -p sessions with the same cwd share ONE database. This means:
- preIndexSharedDocs writes to the same DB as plan-work, build, review
- The CONTEXT-SHARING.md doc incorrectly implies per-session isolation
- Session ID sharing was designed for a problem that doesn't exist

**Problem 1: preIndexSharedDocs wastes a full claude -p session**
The orchestrator spawns an entire Claude session just to Read 11 files into context-mode.
Cost: ~$0.02 + ~30s per auto run.
plan-work Step -0.5 does the same thing with ctx_batch_execute (~2s).
Since they share the same DB, the orchestrator pre-index is 100% redundant.

**Problem 2: Codebase files concatenated into one label**
plan-work Step -0.5 uses:
```
cat .planning/codebase/CODEBASE.md 2>/dev/null || cat ARCHITECTURE.md STRUCTURE.md ...
```
This concatenates 7 files (69KB) into one "CODEBASE" label, losing search granularity.
ctx_search("conventions for tsx") returns the entire 69KB blob instead of just CONVENTIONS.md.

**Problem 3: build doesn't verify stable docs are indexed**
Build Step 2 says "stable docs already indexed from prior step" but has no verification.
If plan-work session failed or context-mode wasn't available, build has no stable context.

**Problem 4: Source files not available to plan-review**
plan-work creates the plan with files_modified list, but doesn't index those source files.
plan-review needs to understand the source to evaluate the plan quality but has to grep/read.

**Problem 5: CONTEXT-SHARING.md has outdated architecture description**
Claims per-session DB isolation via CLAUDE_SESSION_ID. Actually per-project-directory.
</context>

## Task 2: Fix plan-work codebase indexing

In `plan-work/SKILL.md`, update Step -0.5 Phase Context Bootstrap:

Replace the CODEBASE fallback pattern:

With individual file entries:
```
ctx_batch_execute([
  { label: "PROJECT", cmd: "cat .planning/PROJECT.md" },
  { label: "ROADMAP", cmd: "cat .planning/ROADMAP.md" },
  { label: "REQUIREMENTS", cmd: "cat .planning/REQUIREMENTS.md" },
  { label: "DESIGN", cmd: "cat .planning/DESIGN.md 2>/dev/null || echo ''" },
  { label: "ARCHITECTURE", cmd: "cat .planning/codebase/CODEBASE.md 2>/dev/null || cat .planning/codebase/ARCHITECTURE.md 2>/dev/null || echo ''" },
  { label: "STRUCTURE", cmd: "cat .planning/codebase/STRUCTURE.md 2>/dev/null || echo ''" },
  { label: "CONVENTIONS", cmd: "cat .planning/codebase/CONVENTIONS.md 2>/dev/null || echo ''" },
  { label: "TESTING", cmd: "cat .planning/codebase/TESTING.md 2>/dev/null || echo ''" },
  { label: "STACK", cmd: "cat .planning/codebase/STACK.md 2>/dev/null || echo ''" },
], queries: [...])
```

This gives ctx_search per-file granularity: searching "conventions for tsx" returns CONVENTIONS.md content, not all 69KB.

## Task 3: Add post-plan source file indexing to plan-work

After plan-work creates the PLAN.md (end of plan-work flow, before the final summary), add a new step:

```markdown
## Step 9.5: Index Source Files for Downstream Steps

If ctx_batch_execute is available AND the plan has a `files_modified` list in frontmatter:

1. Parse files_modified from the plan frontmatter
2. Filter to files that exist on disk
3. Index them:

ctx_batch_execute([
  ...existingFiles.map(f => ({ label: basename(f), cmd: "cat " + f }))
], queries: ["implementation patterns in modified files"])

This ensures plan-review and build can access the relevant source code via ctx_search
without redundant file reads. If files_modified is empty or ctx_batch_execute is
unavailable, skip silently.
```

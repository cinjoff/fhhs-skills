  <name>Task 1: Server /api/logs endpoint for .auto-log.jsonl</name>
  <files>templates/project-tracker/server.cjs</files>
  <action>
  Add a `/api/logs` endpoint that reads `.auto-log.jsonl` for the active project:

  ```javascript
  app.get('/api/logs', (req, res) => {
    const project = getActiveProject();
    if (!project) return res.json({ entries: [] });
    const logPath = path.join(project.path, '.planning', '.auto-log.jsonl');
    if (!fs.existsSync(logPath)) return res.json({ entries: [] });

    const lines = fs.readFileSync(logPath, 'utf-8').split('\n').filter(l => l.trim());
    const entries = [];
    for (const line of lines) {
      try { entries.push(JSON.parse(line)); } catch { /* skip */ }
    }

    // Support query params for filtering
    const { type, phase, since, limit } = req.query;
    let filtered = entries;
    if (type) filtered = filtered.filter(e => e.type === type);
    if (phase) filtered = filtered.filter(e => String(e.phase) === phase);
    if (since) filtered = filtered.filter(e => e.ts > since);
    const limitN = parseInt(limit) || 500;
    filtered = filtered.slice(-limitN);  // last N entries

    res.json({ entries: filtered, total: entries.length });
  });
  ```

  Also trigger an SSE `refresh` event when `.auto-log.jsonl` is modified (the existing
  fs.watch on `.planning/` should already cover this — verify and add if not).
  </action>
  <verify>Start server, hit /api/logs — confirm it returns parsed JSONL entries. Test with ?type=tool-call filter.</verify>
  <done>Server serves .auto-log.jsonl entries via /api/logs with filtering support.</done>
</task>

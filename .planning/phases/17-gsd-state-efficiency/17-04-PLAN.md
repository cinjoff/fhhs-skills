     - Define three types:
       a) `human-verify` (90% of checkpoints): pause for visual/UX confirmation. User reviews UI, confirms behavior matches design. Example: "Check that the settings page renders correctly."
       b) `decision` (9%): user must choose between options at a fork. Example: "The API returns both formats — should we use JSON or XML?"

     - When to use each type (based on plan task `type` attribute)
     - How build executor handles each:
       - human-verify: present screenshot or description, ask "Does this look correct? [y/n]"
       - decision: present options with trade-offs, wait for choice
       - human-action: present instructions, wait for "done" signal
     - Auto mode behavior: all three types are skipped (auto-decided with best judgment)
  </action>

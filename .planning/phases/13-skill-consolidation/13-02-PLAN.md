<task type="auto">
  <name>Task 2: Add Quality Refinement step to review</name>
  <files>
  .claude/skills/review/SKILL.md
  .claude/skills/review/references/review-prompt.md
  </files>
  <action>
  1. Read review/SKILL.md fully
  2. Add a new "Step 2.5: Quality Refinement" between Step 2 (Dispatch Analysis) and Step 3 (Goal Verification)
  3. Step 2.5 logic — runs after agent findings are collected:

     **Triage findings and dispatch sub-skills as needed:**

     a. Analyze findings from Agent 1 (Code Quality) and Agent 2 (Gap Analysis)
     b. For each finding category, evaluate whether a sub-skill should be dispatched:

     | Finding pattern | Sub-skill | Trigger condition |
     |---|---|---|
     | DRY violations, code duplication, redundant patterns | `/fh:simplify` | 2+ DRY/duplication findings |
     | Unhandled error paths, missing edge cases, brittle patterns | `/fh:harden` | Any unhandled error path in changed code |
     | Cross-device/responsive issues, accessibility gaps | `/fh:adapt` | Frontend files changed + accessibility/responsive findings |
     | Design system drift, inconsistent tokens/spacing | `/fh:normalize` | Frontend files + design system defined in DESIGN.md |
     | Visual quality issues, layout problems, AI slop | `/fh:ui-critique` | Visual file ratio > 30% OR explicit UI concerns in findings |
     | Final polish pass | `/fh:polish` | Only after other sub-skills ran AND findings remain |

     c. Dispatch a SINGLE "quality-refine" subagent (general-purpose) that receives:
        - The findings from Agent 1 and Agent 2
        - The trigger table above
        - The changed file list
        - claude-mem access (pass project name so subagent can use smart_search for cross-session pattern detection)
        The subagent decides which sub-skills to apply based on findings, runs them in sequence, and reports back.
        This keeps review lean (one subagent dispatch, not N sequential inline skill calls).
     d. If no findings trigger any sub-skill: skip entirely (most reviews won't need this)
     e. Sub-skill dispatch is scoped to changed files only — not the whole codebase
     f. If the quality-refine subagent times out or fails, review continues without refinement — log a warning but don't block

     **Performance checks (conditional):**
     - If Next.js project detected (Step 1) AND performance-related findings: reference `.claude/skills/nextjs-perf/PROMPT.md` criteria
     - If frontend changes AND significant bundle/render concerns: suggest `/fh:ui-test` for visual verification

     **Token efficiency:** Use claude-mem smart_search to check if the same quality issues were flagged in prior reviews. If a pattern recurs 3+ times, escalate priority and note in report.

     This step only runs in full mode (not --quick).

  4. Update Step 4 (Evidence Collection):
     - Keep the existing "suggest /fh:ui-test" for frontend work
     - Add note: "If Step 2.5 dispatched sub-skills, re-run tests to verify refinements didn't break anything"

  5. In review-prompt.md, add guidance for Agent 1 to flag which quality refinement categories apply
  </action>

  </verify>
  <done>Review has a Quality Refinement step that conditionally dispatches sub-skills based on findings. Review is now the single quality gatekeeper.</done>

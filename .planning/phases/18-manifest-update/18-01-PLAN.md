  ```
    check: "tool" | "plugin" | "hook" | "env" | "dir" | "file",
    id: string,           // what to check for
    install: string|null, // how to install (null = manual) — validated by validateInstallCommand
    since: string|null,   // version introduced
    deprecated: string|null, // version deprecated
    action: "remove"|null,
    reason: string|null
  }
  ```
  </action>

<task type="tdd">
  <name>Task 2: Manifest engine — check, remediate, discover, generate</name>
  <files>bin/lib/manifest.cjs</files>
  <action>
  Create `bin/lib/manifest.cjs` that absorbs logic from global-reconcile.cjs,
  post-update-reconcile.sh, and changelog.cjs runChecks(). Exports:

  **Reading and checking:**
  - `readManifest(globalPath, projectPath)` — reads both manifest files, falls back to defaults if missing. Returns `{ global, project, merged }`.
  - `checkManifest(merged, projectRoot)` — diffs merged manifest against actual system state. MOVE check logic from changelog.cjs runChecks() here (command -v for tools, installed_plugins.json for plugins, settings.json for hooks/env, fs.existsSync for dirs/files). Each check wrapped in try/catch with 5s timeout.

  **Remediation (absorb from global-reconcile.cjs lines 278-437 and post-update-reconcile.sh):**
  - `remediate(checkResults, projectRoot)` — for each failing item:
    - Tools: install via pnpm/npm (use execFileSync, not shell strings)
    - Plugins: claude plugin install/uninstall
    - Hooks: add/remove from settings.json
    - Env vars: set in global or project settings.json (FHHS_SKILLS_ROOT global, others per-project)
    - Dirs: create with mkdirSync
    - Deprecated+remove: uninstall plugin, remove hook, clean env var
    - Claude-mem settings enforcement (from post-update-reconcile.sh)
    - Tracker registration (from post-update-reconcile.sh)
    Wrap EVERY operation in try/catch. Never throw. Returns `{ remediated[], failed[], removed[], skipped[] }`.

  **Project discovery (absorb from global-reconcile.cjs lines 64-207):**
  - `discoverProjects()` — find all fhhs-skills projects:
    1. Query Conductor SQLite DB for active workspaces (state='ready')
    2. Read ~/.claude/tracker/projects.json (skip Conductor paths — already covered)
    3. Fallback: scan ~/conductor/workspaces/ filesystem
    Dedup by normalized path. Handle: DB unavailable, sqlite3 missing, registry corrupt — all silent fallback.

  **Generation:**
  - `generateManifest(projectRoot)` — auto-detect project state and generate manifest.
    Framework: package.json deps (next→nextjs, react-scripts→react, express→node, etc.)
    Database: deps (supabase, pg, prisma, drizzle)
    Auth: deps (better-auth)
    Testing: config files (vitest.config→vitest, jest.config→jest, playwright.config→playwright)
    Package manager: lock files (pnpm-lock.yaml→pnpm, yarn.lock→yarn, package-lock.json→npm)
    Planning: filesystem checks (.planning/, CLAUDE.md, conductor.json)

  **Orchestrators:**
  - `checkAndRemediate(globalPath, projectPath, projectRoot, opts)` — single-project orchestrator.
    opts: { noRemediate: bool }. Calls read→check→remediate→return JSON.
  - `checkAll(globalPath, opts)` — multi-project orchestrator (replaces global-reconcile main).
    Calls discoverProjects(), then checkAndRemediate per project, groups by repo.
    opts: { noRemediate: bool, changelogFile: string (for --all backward compat) }

  **Output contract (always valid JSON):**
  ```json
  {
    "status": "ok" | "fixed" | "errors",
    "version": "1.61.4",
    "items": [
      { "category": "plugin", "id": "claude-mem", "status": "ok", "details": "Installed" },
      { "category": "plugin", "id": "context-mode", "status": "removed", "details": "Deprecated in 1.62.0" },
      { "category": "hook", "id": "statusline", "status": "fixed", "details": "Was missing, added" },
      { "category": "tool", "id": "node", "status": "error", "details": "Not found", "hint": "Install Node.js 20+ from nodejs.org" }
    ],
    "profile": { "framework": "nextjs", "database": "supabase" },
    "summary": { "ok": 6, "fixed": 1, "removed": 1, "errors": 0, "skipped": 0 },
    "errors": [ { "code": "TOOL_MISSING", "message": "node not found", "hint": "Install Node.js" } ]
  }
  ```

  **Resilience rules:**
  - Every check wrapped in try/catch — failure = status "error" with hint, not a crash
  - If manifest file is corrupt JSON, log warning, fall back to defaults
  - If a remediation command fails, record in failed[] with the error message, continue to next
  - If filesystem paths don't exist, report but don't crash
  - Always return valid JSON — the output function is in a top-level try/catch
  </action>
  <verify>Unit test with mocked filesystem and exec: (1) checkManifest detects a missing tool, (2) remediate attempts install and records result, (3) deprecated item triggers removal, (4) corrupt manifest file falls back to defaults, (5) output is always valid JSON even when operations fail, (6) discoverProjects handles DB unavailable with silent fallback, (7) generateManifest detects Next.js from package.json deps.</verify>
  <done>Engine reads manifests, diffs state, remediates issues, discovers projects, generates manifests, and always produces valid structured JSON with actionable error hints. global-reconcile.cjs logic fully absorbed.</done>
</task>

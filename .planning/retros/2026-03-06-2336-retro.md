### Actionable Learnings

1. **Before adding any setup/config step to a command, grep for the config key in composites first.** The `/settings` revert happened because usage was assumed rather than verified.

2. **When auditing "no external dependencies," explicitly check for `require()`/`exec()` calls to external binaries** — not just npm packages. `gsd-tools.cjs` was missed because it's called via shell, not imported.

3. **Address all items from a multi-part request in order, or explicitly defer with a note.** The README rewrite was part of the original ask but got deprioritized — the user had to see it mentioned at the end before it got done.

4. **The final-wave spec gate gap in `/build` is a known open issue** — if `/build` is ever revisited, consider running a lightweight spec check on the final wave or making the skip configurable.

5. **For future eval runs**: the mock project + `evals.json` are in `fhhs-skills-workspace/`. Rerun by spawning the same 4 subagent prompts against the command files and comparing against the assertions. No new scaffolding needed.

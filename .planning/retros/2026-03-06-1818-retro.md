### Discoveries

- **Claude Code plugin system has no formal dependency field** — `plugin.json` does not support a `dependencies` key; runtime checks via skill preambles are the only viable mechanism.
- **`/install-plugin` is a user-facing slash command**, not a bash-executable command — attempting to automate it via shell would silently fail.
- **500-line soft cap on skill files** — `build.md` at 437 lines was approaching the limit; extracting to `references/` dropped it to 311.

---

### Friction Points

- **Missing requirement upfront: "GSD is always required"** — This should have been established at the start of the session. It caused a large rework cycle (add fallbacks → remove fallbacks) that consumed significant edits across many files.
- **Step renumbering cascades** — Renumbering steps in `build.md` from decimal to sequential required multiple sequential edits because cross-references to step numbers appear elsewhere in the same file. A more systematic search-before-edit would have caught all references in one pass.

---

### Actionable Learnings

1. **Clarify hard constraints before writing** — Before implementing multi-path logic (GSD required vs optional, branch vs no-branch), ask: "Are there any dependencies that are always present?" One question saves many file edits.
2. **Search for cross-references before renumbering** — When renumbering steps, `Grep` for all step number mentions in the file first, then edit all at once. Don't discover them piecemeal.
3. **Use sequential step numbers from day one** — Decimal steps (3.45, 3.5) indicate late insertions. If a step is added mid-process, renumber immediately rather than using decimal shims.
4. **List items must have content** — If a list item ends with a colon (e.g., "These behavioral directives:"), the content must follow immediately in the same list. Dangling colons with content elsewhere in the doc confuse model parsing.
5. **For `/setup`-style commands, always specify what the user must type manually** — Any slash command that installs or configures other tools cannot be automated; the skill must explicitly instruct the user to type it.

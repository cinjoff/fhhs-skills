## Architecture

**Layer model:**
```
Presentation (Server & Client Components)
    ↓
API Routes (src/app/api/)
    ↓
Supabase (database + auth)
```

**Data flow:**
1. Server components fetch data directly or via API routes
2. Client components use `src/lib/api.ts` fetch wrapper for mutations
3. API routes validate input, call Supabase, return JSON responses
4. Auth: JWT tokens via `src/lib/auth.ts` (signToken/verifyToken), session check via `useAuth` hook

**Key boundaries:**
- Presentation never calls Supabase directly — always goes through API routes
- `src/lib/auth.ts` is the single auth utility; hooks consume it
- Type definitions centralized in `src/types/index.ts`
- Supabase client initialized once in `src/lib/supabase.ts`
